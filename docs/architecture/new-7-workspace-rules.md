* *Project‚Äëfirst organization*: Files live under projects, not file types.

* **Single working directory**

  * Example: `~/Work/Active/`
* **Standard project skeleton**

  ```

  ‚îú‚îÄ design/
  ‚îú‚îÄ src/

**Automation ideas**

* Weekly script or rule to move untouched files ‚Üí `_archive/`
* OS search (Spotlight / Windows Search) replaces browsing

## 1Ô∏è‚É£ Decluttering Files by Lifecycle (Not by Location)

![Image](https://technicalwriterhq.com/wp-content/uploads/2022/02/Document-lifecycle-management-1024x622.jpg)

![Image](https://eaa-online.org/app/uploads/sites/3/2019/04/system_files_u271_folder_structure.png)

![Image](https://flow-e.com/wp-content/uploads/2018/04/Inbox-Zero-Outlook-Workflow-Chart.png)

### Use a **file lifecycle model**

Every file should clearly belong to **one of four states**:

1. **Capture** ‚Äì temporary, unsorted (downloads, screenshots)
2. **Active** ‚Äì currently being worked on
3. **Reference** ‚Äì useful but not changing
4. **Archive** ‚Äì completed, immutable

**Implementation**

* One capture location only (Downloads or Desktop)
* Active files live *only* inside active project folders
* Reference files are read-only by convention
* Archive is write‚Äëprotected (mentally and structurally)

**Key rule**

> If a file doesn‚Äôt clearly belong to one state, it‚Äôs friction.

## 2Ô∏è‚É£ Naming as an Index, Not a Label

**High-signal filename pattern**

```

```

Example:

```

```

**Why this works**

* Chronology without folders
* Context survives relocation
* Search > navigation
* Files stay understandable years later

## A. File Decluttering via ‚ÄúActive Surface Minimization‚Äù

![Image](https://www.cs.uic.edu/~jbell/CourseNotes/OperatingSystems/images/Chapter12/12_04_Virtual_FS.jpg)

![Image](https://us1.discourse-cdn.com/flex023/uploads/core77/optimized/3X/9/5/95074786199b31251e87d9673ec0e367e023f270_2_387x500.jpeg)

![Image](https://static0.xdaimages.com/wordpress/wp-content/uploads/wm/2025/05/3-add-folder.png?dpr=2\&fit=contain\&h=422\&q=49\&w=750)

![Image](https://d2xda5fikm7vtb.cloudfront.net/resize%3Dwidth%3A2048%2Cfit%3Amax/Zq2LotSsu9rfsyFrlmkQ)

### Goal

Reduce the *visible surface area* of files competing for attention.

### Strategy

* **One visible workspace only**

  * Everything else is hidden, archived, or searchable.
* **Strict Active/Inactive boundary**

  * Active = touched in the last 7‚Äì14 days
  * Inactive = automatically moved out of sight

### Implementation

* One folder only:

  ```
  /WORKING_NOW/
  ```
* Everything else lives in:

  ```
  /ARCHIVE/YYYY/
  ```
* Downloads, screenshots, exports:

  * Auto-routed to `/INBOX_FILES/`
  * Cleared on a fixed schedule (daily or weekly)

### Rule

> If a file is visible, it must justify its presence *today*.

![Image](https://images.ctfassets.net/lzny33ho1g45/1OxkdcrJXxCq643e6TSib0/924edd006d454a00c1198223a3bf0e7c/organize-files-folders-image3.png)

![Image](https://static0.xdaimages.com/wordpress/wp-content/uploads/wm/2025/05/3-add-folder.png?dpr=2\&fit=contain\&h=422\&q=49\&w=750)

![Image](https://svg.template.creately.com/iquzi3k61)

### Principle

If a file is visible, it is taxing attention.

### System

* **Single active workspace**

  ```
  /ACTIVE/
  ```
* **Everything else is non-visible**

  ```
  /ARCHIVE/YYYY/
  /REFERENCE/
  ```
* **One capture location only**

  * Downloads / Screenshots ‚Üí `/INBOX/`

### Rules

* Files untouched for **7 days ‚Üí auto-archive**
* Desktop icons = **zero**
* No more than **10 files** in `/ACTIVE/`

Result: visibility equals priority.

```
YYYY-MM-DD__context__purpose__state.ext
```

Example:

```

* Search > navigation
* Files remain meaningful outside their folders
* Folder hierarchy becomes shallow and stable
